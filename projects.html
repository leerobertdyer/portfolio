Write A Bad Song
(live site)

A songwriting social media website designed to encourage quantity over quality.
You get points for posting songs, and can collaborate with others.

Highlights
socket.io

Live updates when users submit a post, instant messaging, and notifications for different events were all made possible utilizing socket.io.

Collaboration

Users are able to post lyrics or music alone, and help each other finish songs. Finished collaborations give points to both users.


Pain Points
Firebase Authentication

I initially had my authentication route built using BCRYPT to hash passwords, and cookies to store user information, but quickly hit walls with the method. After several dozens of hours of fixes and patches, I finally realized it was easier and better to let Firebase do the hard work, and as a result my app now functions on mobile devices without jumping through extra hoops.

Dropbox Authentication

I initially thought using dropbox made sense because the site is designed to handle music files, but also profile pictures and other user content. But I've had difficulty with 3rd party cookies using this format, not to mention long load times and issues with registration. I am looking into using firebase storage as an alternative.

Ashvenue
(live site)

A webscrolling app that displays all major Asheville venue events.
Automatically updates, and users can sort by date and venue.

Highlights
Beautiful Soup

Using Beautiful Soup was a breeze, it made scraping the easy part.

SQL Alchemy/Flask

Using flask always feels right, and part of that is all the tools that come with it or work with it, like SQL Alchemy, which helps organize all the data coming in and process it for me.


Pain Points
Date/Time Parsing

Using datetime it was relatively easy to alter the incoming date strings, but some websites used DEC 1st, 2023 while others used 12/1/23 or december 1, 2023 or with no comma or sometimes all caps. It took a lot of finessing to get the date times in an appropriate format for database to be able to sort by date.

Long Waits

I'm still working on a solution for this. I believe multi-threading is the answer, but also looking at ways to break out of the scraping when the information is already in the database, so when it does scrape, it isn't wasting time on redundant data.

Pong Pong
(live site)

A remake of the classic pong game with some twists.
Multiplayer with multiple active games is possible.

Highlights
socket.io

I used sockets to make the game constant for both players. It offers live updates for events like hitting a wall panel or a paddle, or scoring, and updates accordingly.

Node.js Express Server

I keep track of current connected players in a waiting room, and match them to games. Also handles game data and emission of socket events.

P5JS

P5JS was a joy to work with, turning pure javascript into a game was very fun, and being able to see my code immediately on screen was awesome.


Pain Points
Physics

Getting the ball to move in the right direction according to what a user would expect took some solid math skills and a lot of trial and error.

Multiplayer/Multiple Games

Not only was it difficult to establish when to start a game, but also how to have multiple games at once. Once the game functioned this is what I spent the most time working on.
